Swift in Your Browser — No Toolchain Required

We compile a small Swift interpreter to WebAssembly so you can write and run Swift client‑side. No full Swift compiler is bundled, and your code never leaves the page.

Interpreter, Not a Full Toolchain

The experience feels like Swift, but under the hood it’s an interpreter: a lightweight front‑end processes your code and a virtual machine evaluates it inside a WebAssembly module.

Private by Default

Your code runs locally in the browser sandbox. There’s no server round‑trip or remote compilation.

Fast Startup

The WebAssembly payload is small and quick to load — instant run without installing a compiler.

How It WorksHigh‑Level

  1. You write Swift in the editor. When you press Run, your source stays in the browser.
  2. Our lightweight compiler front‑end parses the code into an abstract syntax tree and lowers it to an intermediary format designed for fast evaluation.
  3. A small virtual machine inside the WebAssembly module interprets that intermediary format step‑by‑step and produces results.
  4. A carefully allowlisted host bridge lets your program do safe things (like print output) without exposing the broader environment.

What You Get

  • Core Swift language features: functions, closures, control flow, optionals, collections, string interpolation, and more.
  • Consistent behavior across platforms: the interpreter executes deterministically in the browser sandbox.
  • Small download, instant feedback: no native compiler, no server dependency.

Performance & Safety

  • Optimized interpreter internals (like slot‑based locals and specialization strategies) keep evaluation snappy.
  • Runs entirely in a WebAssembly sandbox; your code and data remain in the page.

This is intentionally an interpreter, not native code generation. That’s how we achieve portability, safety, and instant startup without packaging the full Swift compiler.
